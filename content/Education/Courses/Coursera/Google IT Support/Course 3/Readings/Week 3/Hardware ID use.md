# Hardware ID Use: Implementation and Management

## Purpose and Structure

Hardware IDs are vendor-defined identification strings that Windows uses to match devices with appropriate driver packages. These identifiers:

- Specify what a device is with varying levels of detail
- Indicate which driver packages can work with the device
- Provide a hierarchical matching system from specific to general compatibility

Most devices have multiple hardware IDs organized in order of specificity, from most precise (offering full functionality) to most generic (offering basic functionality).

## Hardware ID Creation

Hardware IDs are generated by a device's enumerator (its bus driver) and reported to the Plug and Play (PnP) Manager. Bus drivers typically use one of these formats:

1. **Standard Device Format**: `<enumerator>\<enumerator-specific-device-ID>`
   - Most common format for individual PnP devices
   - Example: `PCI\VEN_1000&DEV_0001&SUBSYS_00000000&REV_02`

2. **Multi-Enumerator Format**: `*<generic-device-ID>`
   - Asterisk indicates support by multiple enumerators (ISAPNP, BIOS)
   - Example: `root\*PNP0F08`

3. **Device-Class Format**: `<device-class-specific-ID>`
   - Used by established device classes with custom naming conventions

All hardware IDs must be shorter than MAX_DEVICE_ID_LEN characters (excluding NULL terminator).

## Special Considerations for Root-Enumerated Devices

Root-enumerated devices require special attention:

- Created using APIs where hardware IDs can be directly provided
- Generic namespaces like ROOT\SYSTEM may cause conflicts during Windows updates
- Best practice: Use unique namespace format `ROOT\[COMPANYNAME]\[DEVICENAME]`
- Always check if the devnode is already present before installation

## Accessing Hardware IDs

Hardware IDs can be accessed through:

### User Interface Method
1. Open Device Manager
2. Locate the device in the tree
3. Right-click and select Properties
4. Select the Details tab
5. Choose "Hardware Ids" or "Compatible Ids" from the Property dropdown

### Programmatic Method
Use one of these APIs:
- IoGetDevicePropertyData
- SetupDiGetDeviceProperty
- CM_Get_DevNode_Property

The returned data is a REG_MULTI_SZ value with these constraints:
- Maximum characters (including NULL terminators): REGSTR_VAL_MAX_HCID_LEN
- Maximum number of IDs in a list: 64
