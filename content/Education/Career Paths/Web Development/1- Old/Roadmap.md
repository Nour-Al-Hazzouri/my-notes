## What Exactly Did I Do?
I just thought by taking a course for each technology needed, I could become a web developer. Hence the notes for HTML, CSS, etc.

Note: I had Codecademy notes too for HTML, CSS, and JS but deleted them since they were literally copy-paste from the website itself (Yes, unfortunately I thought this was called 'note-taking').

I had simply envisioned that Web Development is in summary:
1. HTML
2. CSS
3. JS
4. React (or any JS-based framework, React is my personal preference)
5. Next.js

So I went to Coursera, found University of Michigan's Web Development Specialization, and started with it, and had a goal to try and complete it and then try to build something.

## Reality Check
This method of online courses is what wasted around 2 or more years of my life attempting to become what I admired.

But the fact is, no matter what you choose in IT, anything that requires development will need some fundamentals—programming is literally the easiest part of that.
- Pseudocode (a.k.a. algorithm)
- Logical Thinking
- Critical Thinking
- Problem Solving
- Data Structures
- Low-Level Programming (Understanding how low-level programming languages work, such as C, saves you a ton of confusion when learning high-level programming, so—NO. Starting with Python was not a good idea)

This is not where the list ends. Even I don't know everything (just knowing, not experiencing or mastering XD), but hitting this realization is a crucial step to understand how to keep going onward.

# How Did I Continue?
I came across a different method of learning, where results aren't determined by an exam or an expected set of logic, **Project-Based Learning.**

**So what exactly is Project-Based Learning?**

Basically, when learning a new topic, instead of a pure theoretical approach, practical experience is prioritized. You learn the concept, and start working on a project to practice it.

My first time experiencing this was with Harvard's CS50x, and boy how hard this course was, yet the most satisfying I ever had.

This made me understand that becoming a developer means understanding the concept, building, breaking, fixing, and then learning.
