- Testing:
    - Confirming that software works as specified in the software’s requirements
- Types of testing:
    - End-to-End (e2e):
        - Test the product just like a user would to make sure it behaves as expected
        - This means that in e2e testing you need to open your web application in a browser and then test it by interacting with the page the same way a user might interact with it.
        - e2e testing frameworks:
            - WebdriverJS
            - Protractor
            - Cypress
    - Integration:
        - Testing how parts of your system interact with other parts of the system
        - Integration testing library:
            - React testing library
            - Enzyme
    - Unit:
        - Process of testing the smallest units of your source code in isolation
        - Unit:
            - Smallest piece of code you can test separately from the rest of the app
            - Usually a function or a method
    - 3-level testing pyramid:
        1. Unit tests
            1. Cheapest and fastest
        2. Integration Tests:
            1. Average speeds and expenses
        3. E2e tests:
            1. Slowest and most expensive
