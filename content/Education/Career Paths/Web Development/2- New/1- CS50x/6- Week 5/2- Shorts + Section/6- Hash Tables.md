
# CS50x Short: Hash Tables

## Introduction to Hash Tables

Hash tables are powerful data structures that provide a way to store and retrieve data with very high efficiency, often approaching constant time (O(1)) for average-case insertions, deletions, and lookups. They achieve this by combining the random access capabilities of arrays with the flexibility of linked lists. The core idea behind a hash table is to use a special function, called a hash function, to map keys (which can be of any data type) to indices in an array. This allows for direct access to the location where the data is stored, eliminating the need for linear traversal.

### Key Components of a Hash Table

A hash table consists of two main components:

1.  **Hash Function:** A function that takes a key as input and returns a non-negative integer, known as a hash code. A good hash function should distribute keys as uniformly as possible across the array to minimize collisions.
2.  **Array (or Hash Table):** An array where the data is stored. The hash code generated by the hash function is used as an index into this array.

### How Hash Tables Work

The process of storing and retrieving data in a hash table involves the following steps:

1.  **Hashing the Key:** When a key-value pair is to be inserted, the key is passed to the hash function, which generates a hash code.
2.  **Mapping to an Index:** The hash code is then mapped to an index in the array, typically using the modulo operator (`%`) with the size of the array. For example, `index = hash_code % array_size`.
3.  **Storing the Data:** The value is stored at the calculated index in the array.

When retrieving data, the same process is followed: the key is hashed, the index is calculated, and the data is retrieved from that index.

### Handling Collisions

A collision occurs when two different keys produce the same hash code, and thus map to the same index in the array. Since it is impossible to store two different values at the same array location, collisions must be handled. The most common method for handling collisions is **separate chaining**.

#### Separate Chaining

In separate chaining, each element of the array is a pointer to a linked list. When a collision occurs, the new key-value pair is simply added to the linked list at that index. This means that instead of storing a single value at each index, the array stores a chain of values that have hashed to the same index.

When inserting a new element:

1.  Hash the key to get an index.
2.  If the linked list at that index is empty, create a new node for the key-value pair and make it the head of the list.
3.  If the linked list is not empty, traverse the list to ensure the key is not already present (if duplicates are not allowed), and then add the new node to the beginning or end of the list.

When searching for an element:

1.  Hash the key to get an index.
2.  Traverse the linked list at that index, comparing the key of each node with the target key.

### Advantages and Disadvantages of Hash Tables

**Advantages:**

*   **Fast Lookups:** On average, hash tables provide constant-time (O(1)) lookups, insertions, and deletions. This is because the hash function allows for direct access to the location of the data.
*   **Flexibility:** They can store a wide variety of data types as keys, as long as a suitable hash function can be defined for them.

**Disadvantages:**

*   **Worst-Case Performance:** In the worst-case scenario (e.g., if all keys hash to the same index), the performance of a hash table can degrade to that of a linked list (O(n) time complexity for lookups, insertions, and deletions).
*   **Not Ordered:** Hash tables do not maintain any order of the elements. If the data needs to be sorted or accessed in a specific order, a hash table is not the appropriate data structure.
*   **Memory Overhead:** The use of linked lists for separate chaining introduces memory overhead for storing pointers.
*   **Hash Function Quality:** The performance of a hash table is highly dependent on the quality of the hash function. A poor hash function that does not distribute keys uniformly can lead to a high number of collisions and degraded performance.

### Applications of Hash Tables

Hash tables are widely used in computer science for various purposes, including:

*   **Implementing Dictionaries and Maps:** They are the primary data structure used to implement associative arrays (or dictionaries) in many programming languages, where keys are mapped to values.
*   **Database Indexing:** Used to quickly locate records in a database based on a key.
*   **Caching:** Used to store frequently accessed data in memory for fast retrieval.
*   **Symbol Tables in Compilers:** Used to store information about variables, functions, and other symbols in a program.

### Conclusion

Hash tables are a highly efficient and versatile data structure for managing key-value pairs. By combining the strengths of arrays and linked lists, they offer excellent average-case performance for lookups, insertions, and deletions. However, their performance is heavily reliant on a good hash function and an effective collision resolution strategy. Understanding the principles of hashing and collision handling is crucial for leveraging the full power of hash tables in various applications.
