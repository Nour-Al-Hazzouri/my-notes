```javascript
{meals.map((meal, index) => (
  <h2 key={index}>{meal}</h2>
))}
```

1. **What `.map()` does**:
   - `.map()` is an array method that creates a new array by calling a function on every element in the original array
   - In React, it's commonly used to render lists of elements

2. **Function Parameters**:
   - The first parameter (`meal`) is the current element being processed
   - The second parameter (`index`) is the position of that element in the array

3. **`index`**:
   - `index` is automatically provided by `.map()`
   - It starts at 0 for the first element
   - For an array `['Breakfast', 'Lunch', 'Dinner']`:
     - First iteration: `meal` = 'Breakfast', `index` = 0
     - Second iteration: `meal` = 'Lunch', `index` = 1
     - Third iteration: `meal` = 'Dinner', `index` = 2

4. **`key` Prop**:
   - In React, when you create lists of elements, each element needs a unique `key`
   - It helps React efficiently update the DOM
   - Ideally, use a unique identifier instead of the index

Example to illustrate:
```javascript
const meals = ['Breakfast', 'Lunch', 'Dinner'];

function MealList() {
  return (
    <div>
      {meals.map((meal, index) => (
        <h2 key={index}>{meal}</h2>
      ))}
    </div>
  );
}
```

This would render:
```html
<div>
  <h2>Breakfast</h2>
  <h2>Lunch</h2>
  <h2>Dinner</h2>
</div>
```

Better `key` practice:
```javascript
const meals = [
  { id: 1, name: 'Breakfast' },
  { id: 2, name: 'Lunch' },
  { id: 3, name: 'Dinner' }
];

function MealList() {
  return (
    <div>
      {meals.map((meal) => (
        <h2 key={meal.id}>{meal.name}</h2>
      ))}
    </div>
  );
}
```

Using a unique `id` is preferred over `index` because:
- It's more stable across re-renders
- Helps React track which items have changed
- Prevents potential rendering issues with list reordering
